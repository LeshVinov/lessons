Действия при первом включении GIT в папке с проектом:

  • git init - инициализация git в папке
  • git config user.name “username” / если будет дополнительно просить выставить name, то используйте команду git config --global user.name “username”
  • git config user.email “email” / если будет дополнительно просить выставить name, то используйте команду git config --global user.email “email”
  • git remote add origin ссылканарепозиторий(GitHub) - подключение репозитория
  • git branch -M main - создание главной ветки main в локальном репозитории (в папке проекта)
  • git add . - добавление в отслеживание git всех файлов в проекте
  • git commit -m “описание сохранения” - создание точки сохранения
  • git push -u origin main - отправка данных в облачный репозиторий (GitHub) и создание в нем ветки main
  • git checkout -b dev - создание ветки dev в локальном проекте

Тут нужно внести какие то данные в ваш проект (любые), что бы появились изменения, которые git увидит

  • git commit -a -m “описание сохранения” - добавление всех файлов в отслеживание (флаг -a) и создание сохранения
  • git push - отправка данных, которая не сработает, поскольку ветки dev в облачном репозитории не существует
  • git push --set-upstream origin dev - отправка данных в облачный репозиторий с созданием в нем ветки dev
  • git checkout -b lesson01 - создание подветки lesson01, исходник для которой - dev
